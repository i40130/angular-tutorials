DIRECTIVAS

Las directivas en Angular son herramientas poderosas que nos permiten extender la funcionalidad del HTML y crear componentes personalizados. Son como mini-componentes que pueden modificar el comportamiento o la apariencia de elementos del DOM.

¿Qué son las directivas en Angular?

Las directivas son clases que añaden un nuevo comportamiento o modifican el existente a los elementos de la plantilla. Básicamente, las directivas se utilizan para manipular el DOM, por ejemplo, añadiendo/quitando 1 elementos del DOM o cambiando la apariencia de los elementos del DOM.

Tipos de directivas

Angular ofrece tres tipos principales de directivas:

  1. Directivas estructurales: Estas directivas modifican la estructura del DOM. Algunos ejemplos comunes son:

    • *ngIf: Muestra o oculta un elemento basado en una condición.
    • *ngFor: Repite un elemento para cada item en una lista.
    • *ngSwitch: Muestra diferentes elementos basados en una expresión.

  2. Directivas de atributos: Estas directivas añaden atributos a elementos existentes y modifican su comportamiento. Algunos ejemplos comunes son:

    • ngClass: Agrega, quita o cambia clases CSS a un elemento.
    • ngStyle: Establece estilos en línea para un elemento.

  3. Directivas de componentes: Los componentes son en realidad directivas con una plantilla. Son la forma más común de crear elementos personalizados en Angular.

Creando una directiva personalizada

Para crear una directiva personalizada, utilizamos el decorador @Directive. Aquí un ejemplo de una directiva que resalta un elemento:

TypeScript

import { Directive, ElementRef, HostListener } from '@angular/core';

@Directive({
  selector: '[appHighlight]'
})
export class HighlightDirective {
  constructor(private el: ElementRef) {
    el.nativeElement.style.backgroundColor = 'yellow';
  }
}

En este ejemplo:

  • @Directive: Indica que esta clase es una directiva.
  • selector: Define el selector que se utilizará en el HTML para aplicar la directiva.
  • HostListener: Se utiliza para escuchar eventos en el elemento host.

Utilizando la directiva en una plantilla

HTML

<p appHighlight>Este párrafo estará resaltado</p>

Otros conceptos importantes

  • Directivas de atributos: Se utilizan para cambiar la apariencia o el comportamiento de un elemento.
  • Directivas estructurales: Se utilizan para modificar la estructura del DOM.
  • Directivas de componentes: Son componentes con una plantilla.
  • HostListener: Se utiliza para escuchar eventos en el elemento host.
  • HostBinding: Se utiliza para enlazar propiedades de la directiva con propiedades del elemento host.

Beneficios de las directivas

  • Reutilización de código: Crea componentes personalizados reutilizables.
  • Mejora la legibilidad del código: Separa la lógica de presentación de la lógica de negocio.
  • Mayor flexibilidad: Permite crear comportamientos personalizados.

Ejemplos más avanzados

  • Creación de validaciones personalizadas: Crear directivas para validar campos de formularios.
  • Creación de componentes drag and drop: Utilizar directivas para implementar funcionalidades de arrastrar y soltar.
  • Creación de componentes de autocompletado: Crear componentes que sugieran opciones a medida que el usuario escribe.
