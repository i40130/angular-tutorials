Rutas Hijas en Angular

Las rutas hijas en Angular son una forma de organizar tu aplicación en una jerarquía, permitiendo una navegación más profunda y una estructura más clara. Son especialmente útiles para crear aplicaciones con múltiples niveles de navegación, como una aplicación de comercio electrónico con categorías y productos.

¿Qué son las rutas hijas?

Imagina que tienes una aplicación de una tienda en línea. La ruta principal podría ser /productos. Dentro de esta ruta, puedes tener rutas hijas como /productos/electronica y /productos/ropa. Estas rutas hijas representan secciones más específicas dentro de la categoría principal de “productos”.

¿Cómo se crean?

  1. Configuración de las rutas:

    • Arreglo de rutas: En tu módulo, define un arreglo de rutas. Dentro de este arreglo, puedes especificar rutas hijas utilizando la propiedad children.
    • Ruta padre: La ruta que contiene las rutas hijas se considera la ruta padre.

    TypeScript

    const routes: Routes = [
  {
    path: 'productos',
    component: ProductosComponent,
    children: [
      { path: 'electronica', component: ElectronicaComponent },
      { path: 'ropa', component: RopaComponent }
    ]
  }
];

  2. Componente padre:

    • RouterOutlet: En el componente padre, utiliza la directiva router-outlet para mostrar el contenido de las rutas hijas.
    • Enlaces: Crea enlaces a las rutas hijas utilizando el routerLink.

    HTML

    <nav>
  <a routerLink="electronica">Electrónica</a>
  <a routerLink="ropa">Ropa</a>
</nav>
<router-outlet></router-outlet>

Ejemplo completo

TypeScript

// app.module.ts
import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: 'productos',
    component: ProductosComponent,
    children: [
      { path: 'electronica', component: ElectronicaComponent },
      { path: 'ropa', component: RopaComponent }
    ]
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppModule { }

Puntos clave a recordar

  • Ruta base: La ruta base de las rutas hijas es relativa a la ruta padre.
  • RouterOutlet: Este elemento en el componente padre actúa como un contenedor para mostrar el contenido de las rutas hijas.
  • Enlaces: Los enlaces a las rutas hijas se crean utilizando el routerLink.
  • Parámetros de ruta: Puedes utilizar parámetros de ruta en las rutas hijas para pasar datos dinámicos.
  • Guards: Los guards se pueden utilizar para proteger el acceso a ciertas rutas hijas.

Beneficios de las rutas hijas

  • Organización: Ayuda a organizar tu aplicación en una estructura jerárquica.
  • Reutilización: Puedes reutilizar componentes en diferentes rutas hijas.
  • Navegación más profunda: Permite una navegación más profunda y detallada dentro de tu aplicación.
  • Mejor experiencia de usuario: Ofrece una experiencia de usuario más intuitiva y consistente.
