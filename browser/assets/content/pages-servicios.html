SERVICIOS

Los servicios en Angular son una pieza clave para la organización y reutilización de la lógica de negocio. Son como cajas de herramientas que contienen funcionalidades específicas que pueden ser compartidas y utilizadas por múltiples componentes.

¿Qué son los servicios en Angular?

Los servicios en Angular son clases TypeScript que encapsulan una funcionalidad específica, como:

  • Acceso a datos: Realizar peticiones HTTP a un servidor.
  • Manejo de estados: Gestionar el estado global de la aplicación.
  • Validación de datos: Validar la entrada del usuario.
  • Utilidades: Proporcionar funciones comunes que pueden ser utilizadas en diferentes partes de la aplicación.

¿Por qué utilizar servicios?

  • Reutilización de código: Evita repetir código en múltiples componentes.
  • Mejor organización: Separa la lógica de negocio de la vista.
  • Facilita las pruebas: Permite probar de forma aislada cada servicio.
  • Mejora la mantenibilidad: Hace que el código sea más fácil de entender y mantener.

¿Cómo crear un servicio?

Para crear un servicio, se utiliza el decorador @Injectable():

TypeScript

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class MiServicioService {
  constructor() { }

  // Métodos del servicio
  saludar(nombre: string) {
    return `Hola, ${nombre}!`;
  }
}

¿Cómo inyectar un servicio en un componente?

Para utilizar un servicio en un componente, se inyecta en el constructor del componente:

TypeScript

import { Component } from '@angular/core';
import { MiServicioService } from './mi-servicio.service';

@Component({
  selector: 'app-mi-componente',
  templateUrl: './mi-componente.component.html',
  styleUrls: ['./mi-componente.component.css']
})
export class MiComponenteComponent {
  constructor(private miServicio: MiServicioService) { }

  saludar() {
    console.log(this.miServicio.saludar('Angular'));
  }
}

Características clave de los servicios

  • Inyección de dependencias: Los servicios pueden depender de otros servicios, lo que permite crear una gráfica de dependencias.
  • Ciclo de vida: Los servicios tienen un ciclo de vida similar al de los componentes.
  • Observables: Los servicios a menudo utilizan observables para manejar datos asíncronos y eventos.
  • Proveedores: Los servicios se proporcionan a los componentes a través de proveedores.

Ejemplos de uso de servicios

  • Servicios HTTP: Para realizar peticiones a una API y obtener datos.
  • Servicios de autenticación: Para gestionar la autenticación de usuarios.
  • Servicios de almacenamiento: Para almacenar datos localmente (localStorage, sessionStorage).
  • Servicios de utilidades: Para proporcionar funciones comunes como validación de formularios, formateo de fechas, etc.
