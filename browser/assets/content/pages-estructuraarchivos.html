ESTRUCTURA DE ARCHIVOS

La estructura de un proyecto Angular es bastante organizada y sigue una convención de nombres que facilita la comprensión y el mantenimiento del código. A continuación, te explicaré los archivos y carpetas más importantes que encontrarás en un proyecto Angular típico generado con Angular CLI:

Estructura Básica de un Proyecto Angular

Cuando creas un nuevo proyecto Angular utilizando ng new my-app, se genera una estructura de carpetas y archivos predeterminada que se organiza de la siguiente manera: 

my-app/
├── e2e/
├── node_modules/
├── src/
│   ├── app/
│   │   ├── app.component.ts
│   │   ├── app.component.html
│   │   ├── app.component.css
│   │   ├── app.module.ts
│   │   └── ... (otros componentes, servicios, etc.)
│   ├── assets/
│   ├── environments/
│   ├── favicon.ico
│   ├── index.html
│   ├── main.ts
│   ├── polyfills.ts
│   └── styles.css
├── .angular.json
├── .editorconfig
├── .gitignore
├── package.json
├── README.md
└── tsconfig.json

Explicación de las carpetas y archivos clave:

  • e2e: Contiene las pruebas end-to-end de la aplicación.
  • node_modules: Almacena todas las dependencias del proyecto, es decir, los paquetes de npm que se instalan.
  • src: Contiene el código fuente de la aplicación.
    • app: Aquí se encuentran los componentes, servicios, módulos y otros elementos principales de la aplicación.
    • assets: Almacena archivos estáticos como imágenes, fuentes, etc.
    • environments: Contiene archivos de configuración para diferentes entornos (desarrollo, producción).
    • index.html: El archivo HTML principal de la aplicación.
    • main.ts: El punto de entrada de la aplicación.
    • polyfills.ts: Contiene polyfills para navegadores que no soportan todas las características de JavaScript modernas.
    • styles.css: Hoja de estilos global para la aplicación.
  • .angular.json: Contiene la configuración del proyecto, como rutas, opciones de compilación, etc.
  • package.json: Describe el proyecto y sus dependencias.
  • tsconfig.json: Contiene la configuración del compilador de TypeScript.

Elementos clave dentro de la carpeta app:

  • Componentes: Cada componente representa una parte de la interfaz de usuario y consta de tres archivos:
    • .ts: Contiene la lógica del componente (TypeScript).
    • .html: Define la plantilla del componente (HTML).
    • .css: Contiene los estilos específicos del componente (CSS).
  • Módulos: Organizan el código de la aplicación en bloques funcionales. El módulo raíz AppModule es el punto de partida de la aplicación.
  • Servicios: Contienen la lógica de negocio de la aplicación y se utilizan para compartir datos y funcionalidad entre componentes.

¿Por qué esta estructura?

Esta estructura proporciona una forma clara y organizada de gestionar un proyecto Angular. Al separar los componentes, servicios y módulos, se facilita la mantenibilidad y la escalabilidad de la aplicación. Además, Angular CLI ofrece herramientas para generar automáticamente nuevos componentes, servicios y módulos, lo que agiliza el desarrollo.
