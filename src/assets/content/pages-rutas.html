RUTAS

Las rutas en Angular son fundamentales para crear aplicaciones de una sola página (SPA) dinámicas y navegables. Permiten que los usuarios se muevan entre diferentes vistas de nuestra aplicación sin realizar recargas completas de página.

¿Qué son las rutas en Angular?

Las rutas en Angular establecen una correspondencia entre las URL y los componentes de nuestra aplicación. Es decir, cada URL se asocia a un componente específico que se mostrará en la pantalla cuando el usuario navega a esa URL.

¿Cómo funcionan las rutas en Angular?

  1. Configuración de las rutas: Se define un arreglo de objetos de configuración de rutas, donde cada objeto representa una ruta.
  2. Módulo de enrutamiento: Este módulo se importa en el módulo principal de la aplicación y se configura con el arreglo de rutas.
  3. RouterOutlet: Este elemento en un componente actúa como un contenedor para mostrar el componente asociado a la ruta activa.
  4. Navegación: Los usuarios pueden navegar entre las rutas haciendo clic en enlaces o utilizando el programador de la aplicación.

Ejemplo básico:

TypeScript

// app-routing.module.ts
import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { HomeComponent } from './home/home.component';
import { AboutComponent } from './about/about.component';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

HTML

<nav>
  <a routerLink="/">Home</a>
  <a routerLink="/about">About</a>
</nav>
<router-outlet></router-outlet>

En este ejemplo:

  • Se definen dos rutas: una para la página de inicio (/) y otra para la página “Acerca de” (/about).
  • Cada ruta se asocia a un componente específico.
  • El router-outlet en el componente principal mostrará el componente correspondiente a la ruta activa.

Conceptos clave adicionales:

  • Rutas hijas: Permiten crear jerarquías de rutas, lo que es útil para organizar aplicaciones grandes.
  • Parámetros de ruta: Se utilizan para pasar datos dinámicos a los componentes.
  • Guards: Protegen rutas, permitiendo o denegando el acceso según ciertas condiciones.
  • Lazy loading: Carga módulos de forma diferida, mejorando el rendimiento de la aplicación.
