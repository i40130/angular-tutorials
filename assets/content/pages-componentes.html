COMPONENTES

Los componentes son el pilar fundamental de las aplicaciones Angular. Son como pequeños bloques de construcción que se combinan para formar la interfaz de usuario.

¿Qué es un componente en Angular?

Un componente en Angular es una clase de TypeScript que encapsula:

  • Una plantilla HTML: Define la estructura visual del componente.
  • Un estilo CSS: Personaliza la apariencia del componente.
  • Lógica de negocio: Maneja los datos, eventos y la interacción del usuario.

¿Para qué sirven los componentes?

  • Reutilización: Un componente puede ser utilizado en múltiples lugares de la aplicación, lo que reduce la duplicación de código.
  • Modularidad: Cada componente se encarga de una parte específica de la interfaz, lo que facilita el desarrollo y mantenimiento.
  • Encapsulación: La lógica de un componente está aislada de otros componentes, lo que mejora la organización del código.
  • Testabilidad: Los componentes son más fáciles de probar de forma aislada.

Estructura básica de un componente

A nivel de archivo un componente está formado por:

  • Archivo HTML (estructura)
  • Archivo CSS (estilos y diseño)
  • Archivo TS (typescript - funcionalidad)
  • Archivo .SPEC.TS (testing)

TypeScript

// mi-componente.component.ts
import { Component } from '@angular/core';
@Component({
selector: 'app-mi-componente',
template: `<p>Hola, mundo desde un componente!/p>`,
styles: [`p {color: blue;} `]
})
export class MiComponenteComponent {}

selector: El nombre que se utilizará en las plantillas HTML para referenciar al componente.
templateUrl: La ruta al archivo HTML que contiene la plantilla del componente.
styleUrls: Un arreglo de rutas a los archivos CSS que contienen los estilos del componente.

Características clave de los componentes

  • Entrada y salida de datos: Los componentes pueden recibir datos desde otros componentes mediante propiedades de entrada (@Input) y enviar datos a otros componentes mediante eventos de salida (@Output).

  • Ciclos de vida: Los componentes tienen un ciclo de vida que define las diferentes etapas por las que pasan desde su creación hasta su destrucción.

  • Directivas: Los componentes pueden utilizar directivas para modificar el comportamiento de los elementos del DOM.

  • Servicios: Los componentes pueden inyectar servicios para acceder a datos o realizar tareas comunes.
