Hello World APP

Tutorial para una aplicación “Hola Mundo” en Angular, explicando cada paso y concepto clave.

Requisitos previos

  1. Node.js y npm (o yarn):

    • Node.js: Descarga la versión LTS (Long Term Support) desde https://nodejs.org/. La versión LTS ofrece mayor estabilidad. Una vez instalado, abre tu terminal y ejecuta node -v y npm -v para verificar las versiones.
    • npm (Node Package Manager): Se instala automáticamente con Node.js. Es el gestor de paquetes que usaremos para instalar Angular CLI y otras dependencias.
    • yarn (alternativa a npm): Si prefieres usar yarn, puedes instalarlo globalmente con npm install -g yarn. Los comandos serán similares a npm, pero con yarn add en lugar de npm install.

  2. Un editor de código (recomendaciones y extensiones):

    • Visual Studio Code (VS Code): Recomendado por su gran soporte para Angular. Descárgalo desde https://code.visualstudio.com/.
    • Extensiones recomendadas para VS Code:
      • Angular Language Service: Proporciona autocompletado, detección de errores y navegación para código Angular.
      • TSLint (o ESLint con configuración para TypeScript): Para el análisis estático de código y el cumplimiento de estándares.
      • Prettier - Code formatter: Para formatear el código automáticamente y mantener un estilo consistente.

Pasos

  1. Instalar Angular CLI (Interfaz de Línea de Comandos):

    • Abre tu terminal o línea de comandos (en Windows, puedes usar PowerShell o la Terminal de Windows).
    • Ejecuta: npm install -g @angular/cli
    • El flag -g (global) instala Angular CLI para que esté disponible en cualquier directorio de tu sistema.
    • Para verificar la instalación, ejecuta ng version. Deberías ver información sobre la versión de Angular CLI y otras dependencias.

  2. Crear un nuevo proyecto Angular:

    • Navega a la carpeta donde quieres crear tu proyecto usando el comando cd (change directory). Por ejemplo: cd Documentos/Proyectos.
    • Ejecuta: ng new hola-mundo
    • hola-mundo es el nombre del proyecto. (Puedes cambiarlo).
    • Preguntas de Angular CLI durante la creación:
      • Would you like to add Angular routing?: Para este ejemplo, selecciona “No”. El enrutamiento se utiliza para la navegación entre diferentes páginas dentro de la aplicación.
      • Which stylesheet format would you like to use?: Selecciona “CSS”. Hay otras opciones como SCSS, Sass y Less, que son preprocesadores de CSS que ofrecen características adicionales.

  3. Navegar al directorio del proyecto:

    • Ejecuta: cd hola-mundo

  4. Ejecutar la aplicación:

    • Ejecuta: ng serve --open
    • ng serve compila la aplicación y la sirve en un servidor de desarrollo local.
    • --open abre automáticamente la aplicación en tu navegador predeterminado.
    • La aplicación estará disponible en http://localhost:4200/. Si el puerto 4200 está en uso, Angular CLI buscará automáticamente otro puerto disponible.

  5. Modificar el componente principal:

    • Abre el archivo src/app/app.component.html en tu editor de código.
    • Estructura básica de un componente Angular: Un componente en Angular se compone de:
      • Template (HTML): Define la vista del componente (el HTML que se muestra en el navegador). app.component.html
      • Clase (TypeScript): Contiene la lógica del componente (datos y métodos). app.component.ts
      • Estilos (CSS/SCSS/etc.): Define los estilos visuales del componente. app.component.css (o el formato que hayas elegido)
    • Reemplaza el contenido de app.component.html con:

    HTML

    <h1>{{ title }}</h1>
<p>¡Hola Mundo desde Angular!</p>
    • Abre el archivo src/app/app.component.ts.
    • Modifica la clase AppComponent para agregar una propiedad title:

    TypeScript

    import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'Mi primera app en Angular';
}
    • {{ title }} en el template se llama interpolación. Muestra el valor de la propiedad title de la clase AppComponent.

  6. Guardar los cambios: Guarda ambos archivos. El navegador se actualizará automáticamente mostrando “Mi primera app en Angular” como título y “¡Hola Mundo desde Angular!” como párrafo.

Explicación de archivos y conceptos clave (por si no conoces Angular y te has saltado la primera parte de este tutorial)

  • src/app/app.component.html: La plantilla HTML del componente raíz.
  • src/app/app.component.ts: La clase del componente raíz, donde se define la lógica y los datos.
  • src/app/app.module.ts: El módulo raíz de la aplicación. Declara los componentes que pertenecen a este módulo.
  • main.ts: El punto de entrada principal de la aplicación. Arranca el módulo raíz (AppModule).
  • index.html: El archivo HTML principal que carga la aplicación Angular. Contiene la etiqueta <app-root>, que es el selector del componente raíz.
  • angular.json: Archivo de configuración del proyecto Angular.
  • package.json: Archivo que contiene información sobre el proyecto y sus dependencias (paquetes npm).
  • Componentes: Son los bloques de construcción fundamentales de las aplicaciones Angular. Encapsulan la lógica, la vista y los estilos.
  • Directivas: Permiten modificar el DOM (Document Object Model) de diferentes maneras.
  • Servicios: Clases que encapsulan la lógica de negocio y se pueden inyectar en los componentes.
  • Módulos: Agrupan componentes, directivas y servicios relacionados.
